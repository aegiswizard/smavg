# Smavg Contributor Terms

These terms explain how contributions are accepted.

By submitting a pull request, patch, issue comment containing code, benchmark,
documentation, test, example, or other project material, you agree that your
contribution is provided under:

```text
Smavg Modified MIT License, Version Genesis 1.0 2026
```

## What This Means

- Your contribution can be used, modified, distributed, sublicensed, and sold as
  part of Smavg under the project license.
- Your contribution may be included in future Smavg releases.
- You confirm you have the right to submit the contribution.
- You should not submit secrets, private data, copyrighted third-party material
  without permission, or code you cannot license to the project.

## Attribution

Contributors may be credited in project history, release notes, or contributor
files when appropriate.

Submitting a contribution does not transfer ownership of the Smavg name,
branding, project identity, or official project direction.

## No Secrets

Do not submit:

- API keys
- passwords
- private user data
- private benchmark corpora
- proprietary files unless you have clear rights to publish them

## License Consistency

If you need different contribution terms, discuss them before submitting.
