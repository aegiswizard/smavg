# Smavg Legal Notes

These notes explain the intent behind Smavg licensing. They are not legal
advice and do not replace [LICENSE](LICENSE).

## License Type

Smavg uses a custom MIT-style permissive license:

```text
Smavg Modified MIT License, Version Genesis 1.0 2026
```

It is designed to preserve broad freedom to use, modify, fork, distribute, and
commercialize Smavg while protecting origin attribution.

## Open-Source Position

Smavg is source-available and built in the spirit of open-source collaboration.

Because the license is custom and includes commercial-scale attribution, it
should not be described as plain MIT. It also should not be described as
OSI-approved unless the license is submitted to and approved by the Open Source
Initiative in the future.

Use this wording:

```text
Smavg uses a custom MIT-style permissive license with commercial-scale attribution.
```

Avoid this wording:

```text
Smavg is plain MIT.
Smavg is OSI-approved.
```

## Commercial Scale

Commercial scale does not create an automatic payment obligation.

The default obligation at scale is visible attribution:

```text
Powered by Smavg 🐲 by Aegis Wizard 🧙‍♂️
```

Commercial licensing is optional for organizations that want different terms,
such as attribution-free use, dedicated support, legal indemnity, or custom
deployment terms.

## Code License Versus Brand

The code license gives broad rights to use the software.

It does not give permission to misrepresent a fork as official Smavg, imply
endorsement by Roman Mataru or Aegis Wizard, or use Smavg branding in a
confusing way.
