# Smavg Context v1 Report

Generated on 2026-05-11.

## What Changed

Smavg now has a second direction besides archive compression:

- `smavg context SOURCE --out context.md --json context.json`
- `smavg expand-context context.json RELATIVE_PATH --out exact-file`

`context` builds a deterministic AI-readable repetition map for a folder. It
does not ask an AI model to summarize or regenerate files. It reports measured
facts: file counts, token estimates, repeated families, common headings,
repeated literal lines, variable markers, largest exact files, SHA-256 roots,
and exact retrieval commands.

`expand-context` reads the context JSON, locates the current source file,
verifies size and SHA-256, and writes the exact bytes to the requested output.

This is the first implementation of Smavg as a repetition firewall for AI
agents.

## Trust Boundary

Smavg Context v1 is intentionally factual:

- It estimates tokens deterministically; it does not claim tokenizer-perfect
  accounting for any one model.
- It reports repeated structure, paths, sizes, headings, line repetition, and
  hashes.
- It does not produce semantic explanations such as "this subsystem was
  refactored for performance" unless that wording is present in file text.
- Exact file content must be retrieved through `expand-context` when citations,
  line-level facts, or byte-perfect content matters.

This makes the context brief safe for AI agents: the model gets the map first
and asks for exact coordinates only when needed.
