# Smavg MCP Server v1

Date: 2026-05-16

This pass exposes Smavg's verified CLI/core as a minimal MCP stdio server so
MCP-compatible AI agents can use Smavg as a repetition firewall.

## Scope

This is intentionally a thin layer. It does not replace the CLI and does not
add new compression behavior. It exposes the already verified local routines:

- preflight
- context
- exact context expansion
- workflow profile listing
- `.smavg` archive verification
- `.smavg` archive reporting

## Code Changes

- Added `src/smavg/mcp_server.py`.
- Added CLI command:
  `smavg mcp-server`.
- Added package script:
  `smavg-mcp = "smavg.mcp_server:main"`.
- Added tests:
  `tests/test_mcp_server.py`.

The server is dependency-free and speaks newline-delimited JSON-RPC over stdio.
It implements:

- `initialize`
- `ping`
- `tools/list`
- `tools/call`
- `notifications/initialized`

Tool-originated errors are returned as tool results with `isError: true` so
agents can see and correct normal Smavg failures such as missing source paths.

## Tools

| Tool | Purpose |
| --- | --- |
| `smavg_preflight` | Create a timestamped preflight for a workflow or source folder. |
| `smavg_context` | Build a deterministic context report for a folder. |
| `smavg_expand` | Expand one exact file from context JSON with SHA-256 verification. |
| `smavg_workflows` | List known workflow profiles. |
| `smavg_verify_archive` | Verify a `.smavg` archive. |
| `smavg_report_archive` | Return a structured `.smavg` archive report. |

## Interpretation

MCP v1 is the first universal-agent surface for Smavg. Codex remains the build
environment, but any MCP-compatible agent can now discover and call the same
preflight/context/expand behavior once the server is configured in that agent.

The trust boundary is unchanged: Smavg gives agents a compact map first, then
exact files on demand. It does not ask AI to regenerate exact bytes.
