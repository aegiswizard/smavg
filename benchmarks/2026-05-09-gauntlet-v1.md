# 2026-05-09 Smavg Real-Data Gauntlet v1

This is the first staged Smavg gauntlet run across local Mac data, existing
Smavg proof corpora, and public reproducible corpora. It uses the normal
Container v1 CLI path: pack, verify, restore, and regular-file byte diff.

Command:

```bash
cd /Users/mac/smavg
PYTHONPATH=src python3 -m smavg.cli gauntlet \
  --preset all \
  --out /tmp/smavg-gauntlet-2026-05-09-v2 \
  --baselines thorough
```

Full raw local report and JSON were saved locally under the ignored folder:

```text
benchmarks/private-local/2026-05-09-gauntlet-v1/
```

That private folder is intentionally not meant for public release because it
contains local machine paths and local file names.

## Counting Rules

`Counted` means Smavg packed the corpus, verified the archive, restored it, and
every regular file restored byte-perfect with the same relative path. If any
filesystem entry was skipped or unsupported, the corpus is not counted.

`Full fidelity` is stricter. It is `NO` when Container v1 restores regular-file
bytes but does not preserve metadata that can matter for a working tree, such as
executable bits, symlinks, hard links, or empty directories.

Baselines are strong solid tar compression:

- `tar | xz -9e`
- `tar | zstd -19 --long=31`
- `tar | brotli -q 11`

## Interpretation

The strongest Smavg claim still holds: versioned-history corpora are the
current world-class category. The extracted Luau history archive was
byte-perfect and 548.438x smaller, beating `xz -9e` by about 19.8%. The mixed
Luau history corpus was also byte-perfect and slightly beat `xz`.

The broad-data result is honest and useful: Smavg often restores real local and
public corpora perfectly, and several categories exceed 10x versus original
size, but general compressors still beat Smavg on many non-history corpora.
That means the next compression work should target logs, JSON, CSV, and
metadata/full-fidelity support rather than adding surface-area features.

## Gaps Found

- Container v1 does not preserve executable mode bits.
- Container v1 does not represent symlinks.
- Container v1 does not represent empty directories.
- Some local code/tooling trees are therefore not full-fidelity archives yet.
- Public Loghub and NVD corpora crossed 10x versus original size but did not
  beat the best strong baseline.

## Next Engineering Target

The next real leap should be full-fidelity tree support before public trust
claims expand beyond regular-file byte content:

1. Preserve directory entries, including empty directories.
2. Preserve executable mode bits.
3. Represent symlinks safely without following them.
4. Add a full-tree restore verifier that checks metadata scope explicitly.
5. Re-run this same gauntlet and promote more corpora from `Counted` to
   `Full fidelity`.
